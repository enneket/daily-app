# 发布指南

## 发布流程

### 1. 确认代码状态

```bash
git status
git log --oneline -3
```

### 2. 更新版本号

编辑 `package.json` 中的 version 字段（如 `1.0.4`）

### 3. 提交更改

```bash
git add .
git commit -m "Release v1.0.4"
```

### 4. 创建并推送 tag

```bash
git tag v1.0.4
git push origin main
git push origin v1.0.4
```

### 5. 等待 GitHub Actions 构建

访问 https://github.com/你的用户名/daily-app/actions 查看构建状态。

构建完成后会自动创建 Release。

## 版本号规范

遵循语义化版本（Semantic Versioning）：

- **主版本号**：不兼容的 API 修改
- **次版本号**：向下兼容的功能性新增
- **修订号**：向下兼容的问题修正

示例：
- `1.0.0` - 首次正式发布
- `1.0.1` - 修复 Bug
- `1.1.0` - 新增功能
- `2.0.0` - 重大更新
