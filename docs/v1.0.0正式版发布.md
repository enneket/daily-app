# v1.0.0 正式版发布

## 🎉 发布说明

日报桌面应用 v1.0.0 正式版发布！这是第一个稳定版本，提供完整的日报记录和展示功能。

## ✨ 核心功能

### 1. 桌面应用

- **跨平台支持**：支持 Windows、macOS、Linux
- **GitHub 集成**：直接提交日报到 GitHub 仓库
- **实时编辑**：Markdown 编辑器，支持实时预览
- **自动保存**：配置信息本地保存
- **版本管理**：自动检测和更新日报仓库文件

### 2. 日报展示网站

- **精美界面**：基于 VitePress 构建的现代化界面
- **日历视图**：按月查看所有日报
- **归档浏览**：按年月分组浏览历史日报
- **统计分析**：自动统计日报数据（总数、连续天数、字数等）
- **全文搜索**：快速查找日报内容
- **自动部署**：GitHub Actions 自动构建和部署

### 3. 智能功能

- **自动初始化**：首次使用自动创建所有必需文件
- **版本检测**：自动检测并更新过期文件
- **文件迁移**：自动迁移旧版本的日报文件
- **路径修复**：自动修复链接路径问题

## 🚀 快速开始

### 1. 下载安装

从 [Releases](https://github.com/你的用户名/daily-app/releases/tag/v1.0.0) 页面下载对应平台的安装包。

### 2. 配置 GitHub

1. 创建 GitHub Personal Access Token
   - 访问：https://github.com/settings/tokens
   - 点击 "Generate new token (classic)"
   - 勾选 `repo` 权限
   - 生成并复制 token

2. 创建日报仓库
   - 在 GitHub 创建新仓库（如 `daily`）
   - 可以是公开或私有仓库

### 3. 配置应用

1. 打开应用，进入"设置"页面
2. 填写配置信息：
   - GitHub Token
   - 仓库所有者（你的 GitHub 用户名）
   - 仓库名称
   - 分支名称（通常是 `main`）
3. 点击"保存并初始化"

### 4. 开始写日报

1. 在编辑器中写日报（支持 Markdown）
2. 点击"提交日报"
3. 等待提交成功

### 5. 查看网站

1. 启用 GitHub Pages（Settings → Pages → Source: GitHub Actions）
2. 等待构建完成（2-5 分钟）
3. 访问：`https://你的用户名.github.io/仓库名/`
