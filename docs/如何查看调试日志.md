# 如何查看调试日志

## 开发模式下查看日志

### 1. 启动开发模式
```bash
npm run dev:electron
```

### 2. 打开开发者工具
应用启动后，开发者工具会自动打开。如果没有自动打开，可以：
- 按 `F12` 键
- 或者按 `Ctrl+Shift+I` (Linux/Windows) / `Cmd+Option+I` (Mac)
- 或者在应用菜单中选择"查看" → "开发者工具"

### 3. 查看控制台日志
1. 在开发者工具中点击 **Console** 标签页
2. 在这里可以看到所有的调试输出，包括：
   - GitHub 服务初始化信息
   - 提交前状态检查
   - shouldAutoPush() 的详细决策过程
   - 本地缓存更新情况
   - 推送条件判断结果

### 4. 测试第一次提交
1. 在应用中输入一些日报内容
2. 点击"提交"按钮
3. 观察控制台输出，特别关注：
   ```
   === 提交前状态检查 ===
   当前 pendingCommits: 0
   当前 lastPushTime: undefined
   ...
   === shouldAutoPush 详细检查 ===
   条件1检查 (累积提交数):
     - 1 >= 10 = false
   条件2检查 (时间间隔):
     - lastPushTime 为空，跳过时间检查（首次使用）
   ❌ 不满足任何推送条件，保持本地暂存
   ```

## 生产模式下查看日志

### 方法1：临时启用开发者工具
在 `src/main/main.ts` 中临时添加：
```typescript
// 在生产模式下也打开开发者工具（仅用于调试）
mainWindow.webContents.openDevTools();
```

### 方法2：查看日志文件
图标相关的调试日志会写入到：
- Windows: `%APPDATA%/daily-report-app/icon-debug.log`
- macOS: `~/Library/Application Support/daily-report-app/icon-debug.log`
- Linux: `~/.config/daily-report-app/icon-debug.log`

## WSL 环境注意事项

如果在 WSL 环境中遇到 X11 显示问题：
```
Missing X server or $DISPLAY
```

需要：
1. 安装 X11 服务器（如 VcXsrv 或 X410）
2. 设置 DISPLAY 环境变量：
   ```bash
   export DISPLAY=:0
   ```
3. 或者使用 Windows 原生环境进行测试

## 常见调试场景

### 场景1：第一次提交直接推送
查看日志中的：
- `pendingCommits` 是否正确从 0 变为 1
- `shouldAutoPush()` 返回值是否为 false
- 是否有意外的推送触发

### 场景2：累积提交不推送
查看日志中的：
- `pendingCommits` 是否正确累积
- 达到 10 个时是否触发推送
- `BATCH_SIZE` 配置是否正确

### 场景3：定时推送不工作
查看日志中的：
- `lastPushTime` 是否正确记录
- 时间差计算是否正确
- `AUTO_PUSH_INTERVAL` 配置是否正确
