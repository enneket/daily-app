# 修复日志刷屏问题

## 问题描述
调试日志在界面上不断刷屏，导致无法正常使用应用和查看有效的调试信息。

## 问题原因
1. **无限循环渲染**：在组件函数体中直接调用 `console.log` 导致每次渲染都输出日志
2. **useEffect 依赖项问题**：`useEffect` 的依赖项包含了会频繁变化的函数，导致重复执行
3. **状态更新循环**：`addDebugInfo` 函数在 `useEffect` 中被调用，更新状态后又触发重新渲染

## 解决方案

### 1. 移除循环触发的日志
- 移除组件函数体中的直接 `console.log` 调用
- 添加 `initialized` 状态来控制初始化日志只执行一次

### 2. 优化 useEffect 依赖项
- 将初始化逻辑分离到独立的 `useEffect` 中
- 移除会导致重复执行的依赖项 `onTodayReportUpdate` 和 `onCommitStatusUpdate`
- 使用空依赖数组 `[]` 确保定时器只设置一次

### 3. 改进调试信息管理
- 优化 `addDebugInfo` 函数，避免不必要的状态更新
- 保持最近 10 条调试信息的显示
- 添加时间戳便于追踪

## 具体修改

### 前端组件优化
```typescript
// 添加初始化状态控制
const [initialized, setInitialized] = useState(false);

// 分离初始化逻辑
useEffect(() => {
  if (!initialized) {
    const timestamp = new Date().toLocaleTimeString();
    const debugLine = `[${timestamp}] 🎯 MainEditor 组件已挂载`;
    console.log(debugLine);
    setDebugInfo([debugLine]);
    setInitialized(true);
  }
}, [initialized]);

// 优化定时器设置
useEffect(() => {
  // 定时器逻辑...
}, []); // 空依赖数组
```

## 修复效果
- 消除了日志刷屏问题
- 保持了必要的调试信息显示
- 提高了应用的性能和稳定性
- 便于用户正常使用和调试

## 测试方法
1. 使用最新打包的应用
2. 应用启动后应该只显示一条初始化日志
3. 进行提交操作时才会显示相关的调试信息
4. 调试信息应该保持在合理的数量范围内
