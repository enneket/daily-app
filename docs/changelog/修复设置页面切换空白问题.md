# 修复设置页面切换空白问题

## 问题描述

用户反馈从设置页面切换回主页面时会有短暂的空白，影响用户体验。

## 根本原因

### 组件重新挂载导致的空白期

当从设置页面切换回主页面时：
1. `MainEditor` 组件会重新挂载
2. 组件挂载时触发 `useEffect` 中的数据加载
3. 在数据加载完成之前，页面显示空白
4. 数据加载包括：
   - 今日已提交内容 (`getTodayReport`)
   - 提交状态信息 (`getCommitStatus`)

### 异步操作阻塞渲染

```typescript
useEffect(() => {
  // 这些异步操作在完成前会导致空白显示
  initializeToday();
  loadCommitStatus();
  // ...
}, []);
```

## 解决方案

### 1. 添加加载状态管理

```typescript
const [loading, setLoading] = useState(true); // 添加加载状态
```

### 2. 优化数据加载流程

```typescript
useEffect(() => {
  const initialize = async () => {
    try {
      // 并行加载数据以提高速度
      await Promise.all([
        initializeToday(),
        loadCommitStatus()
      ]);
    } catch (error) {
      console.error('初始化失败:', error);
    } finally {
      setLoading(false); // 数据加载完成，取消加载状态
    }
  };
  
  initialize();
  // ...
}, []);
```

### 3. 添加加载状态 UI

```typescript
return (
  <div className="flex flex-col h-full">
    {loading ? (
      // 加载状态显示
      <div className="flex items-center justify-center h-full bg-gray-50">
        <div className="text-center">
          <div className="inline-block animate-spin rounded-full h-8 w-8 border-b-2 border-blue-600 mb-4"></div>
          <div className="text-gray-600">加载中...</div>
        </div>
      </div>
    ) : (
      // 主要内容
      <>
        {/* Header */}
        {/* ... 其他组件 */}
      </>
    )}
  </div>
);
```

## 实现细节

### 加载状态管理

1. **初始状态**：`loading = true`
2. **数据加载期间**：显示加载指示器
3. **加载完成**：`setLoading(false)`，显示主要内容

### 并行数据加载

使用 `Promise.all()` 并行加载数据，提高加载速度：

```typescript
await Promise.all([
  initializeToday(),    // 加载今日内容
  loadCommitStatus()    // 加载提交状态
]);
```

### 加载 UI 设计

- **居中布局**：使用 flexbox 居中显示
- **旋转动画**：CSS 动画提供视觉反馈
- **一致的样式**：与应用整体风格保持一致

## 用户体验改进


### 修复后

### 改进效果

1. **消除空白**：不再有空白页面显示
2. **视觉反馈**：用户知道应用正在加载

## 性能优化

### 并行加载
- 使用 `Promise.all()` 并行执行异步操作
- 减少总加载时间

### 缓存利用
- 利用现有的本地缓存机制
- 避免不必要的网络请求

### 渐进式渲染
- 先显示加载状态
- 数据准备好后一次性渲染完整界面
