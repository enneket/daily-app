# 修复一次提交出现两个记录问题

## 问题描述

用户反馈在提交日报时，一次提交出现了两个记录，导致内容重复显示。

## 根本原因

### 1. 时间戳精度不足
当前时间戳只精确到分钟，如果用户在同一分钟内多次提交（如快速双击），会产生相同的时间戳。

### 2. 缺少重复检查机制
代码没有检查是否已存在相同时间戳的记录，直接追加新内容。

### 3. 并发提交问题
后端没有并发保护，理论上可能存在竞态条件。

## 解决方案

### 1. 提高时间戳精度
将时间戳精度从分钟提升到秒，大大减少重复概率。

### 2. 添加重复内容检查
检查是否已存在相同时间戳的记录，如果存在则跳过提交。

### 3. 添加提交锁机制
使用 `isSubmitting` 标志防止并发提交。

### 4. 优化前端防抖
增强前端的重复提交检查。

## 实现细节

### 后端修改
- 新增 `isSubmitting` 提交锁
- 新增 `isDuplicateContent()` 重复检查方法
- 修改 `submitReport()` 方法，添加防重复逻辑
- 时间戳精度提升到秒级

### 前端修改
- 优化 `handleSubmit()` 方法的防抖逻辑
- 添加更严格的重复提交检查

## 防重复机制

### 多层防护
1. 前端防抖：`submitting` 状态防止快速重复点击
2. 后端锁机制：`isSubmitting` 防止并发调用
3. 时间戳精度：精确到秒，减少重复概率
4. 内容检查：检查是否已存在相同时间戳的记录
