# 优化页面切换时的数据加载策略

## 问题描述

从设置页面切换回主页面时，今日内容会重新加载，这是不必要的，因为数据没有变化。用户体验不佳，且浪费网络资源。

## 根本原因

### 组件生命周期问题

```
设置页面 → 主页面
    ↓
MainEditor 组件重新挂载
    ↓
useEffect 触发
    ↓
loadTodayReport() 被调用
    ↓
不必要的数据重新加载
```

### 状态管理问题

- **状态位置**：今日内容和提交状态在 `MainEditor` 组件中管理
- **组件销毁**：页面切换时组件被销毁，状态丢失
- **重复加载**：组件重新挂载时重新加载相同数据

## 解决方案：状态提升

### 核心思路

将数据状态从子组件 `MainEditor` 提升到父组件 `App.tsx` 中管理，这样页面切换时状态不会丢失。

### 实现架构

```
App.tsx (状态管理)
├── todayReport (今日内容)
├── commitStatus (提交状态)
├── dataLoaded (数据加载标志)
└── 数据加载方法
    ↓
MainEditor.tsx (UI 展示)
├── 接收 props
├── 处理用户交互
└── 通过回调更新父组件状态
```

## 实现细节

### 1. App.tsx 状态管理

```typescript
function App() {
  // 提升的状态
  const [todayReport, setTodayReport] = useState('');
  const [commitStatus, setCommitStatus] = useState<CommitStatus>({...});
  const [dataLoaded, setDataLoaded] = useState(false);

  // 只在配置加载完成且数据未加载时加载数据
  useEffect(() => {
    if (config && !dataLoaded && !showSettings) {
      loadInitialData();
    }
  }, [config, dataLoaded, showSettings]);

  const loadInitialData = async () => {
    try {
      await Promise.all([
        loadTodayReport(),
        loadCommitStatus()
      ]);
    } finally {
      setDataLoaded(true);
    }
  };
}
```

### 3. 数据加载时机优化

#### 修改前（每次组件挂载都加载）
```typescript

```

#### 修改后（只在必要时加载）
```typescript
// App.tsx 中
useEffect(() => {
  if (config && !dataLoaded && !showSettings) {
    loadInitialData();  // 只在数据未加载时执行
  }

```

## 数据流设计

### 初始加载流程
```
App 启动 → 加载配置 → 检查 dataLoaded → 加载数据 → 设置 dataLoaded = true
```

### 页面切换流程
```
主页面 → 设置页面 → 主页面
   ↓         ↓         ↓
状态保持   状态保持   状态保持（无重新加载）
```

### 数据更新流程
```
用户提交 → MainEditor.handleSubmit → onDataUpdate → App.handleDataUpdate → 更新状态
```

主要改进：
1. **状态提升**：将数据状态管理移到 App.tsx
2. **智能加载**：只在必要时加载数据
3. **性能优化**：消除重复的网络请求
4. **用户体验**：页面切换更快更流畅
