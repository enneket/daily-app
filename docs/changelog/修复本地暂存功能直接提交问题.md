# 修复本地暂存功能直接提交问题

## 问题描述

用户反馈本地暂存功能仍有问题，提交时还是会直接提交到 GitHub，而不是按预期暂存到本地。

## 根本原因

### 主要问题：getCommitStatus() 方法的错误默认值

```typescript
// 问题代码
getCommitStatus(): { pendingCommits: number; lastPushTime: number; nextPushTime: number } {
  const pendingCommits = this.store.get('pendingCommits') || 0;
  const lastPushTime = this.store.get('lastPushTime') || Date.now(); // ❌ 错误的默认值
  const nextPushTime = lastPushTime + this.AUTO_PUSH_INTERVAL;
  
  return {
    pendingCommits,
    lastPushTime,
    nextPushTime
  };
}
```

**问题分析**：
- `getCommitStatus()` 方法中 `lastPushTime` 使用了 `Date.now()` 作为默认值
- 虽然这个方法主要用于前端显示，但可能影响其他逻辑
- 与 `shouldAutoPush()` 方法中的逻辑不一致

### 次要问题：缺少调试信息

当前代码缺少详细的调试日志，难以定位问题：
- 无法看到 `shouldAutoPush()` 的判断过程
- 无法确认存储状态是否正确
- 无法追踪提交流程

## 解决方案

### 1. 修复 getCommitStatus() 方法

```typescript
// 修复后
getCommitStatus(): { pendingCommits: number; lastPushTime: number; nextPushTime: number } {
  const pendingCommits = this.store.get('pendingCommits') || 0;
```
