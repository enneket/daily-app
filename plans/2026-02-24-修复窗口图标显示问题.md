# 修复窗口图标显示问题

## 问题描述
在 Windows 打包后的应用中，窗口标题栏和任务栏显示的是通用文档图标，而不是自定义的应用图标。

## 问题分析
1. 当前代码在 `main.ts` 中设置了 `icon` 参数，但路径可能不正确
2. 开发模式下图标路径与打包后的路径不同
3. **关键问题**：`package.json` 的 `build.files` 配置中没有包含 `build` 目录，导致打包时图标文件没有被包含进 asar 包中
4. 需要根据运行环境动态设置正确的图标路径

## 解决方案
1. 在 `package.json` 的 `build.files` 中添加 `build/**/*`，确保图标文件被打包
2. 修改 `src/main/main.ts` 中的图标路径逻辑，区分开发模式和生产模式
3. 使用 `nativeImage` API 加载图标
4. 为不同平台设置正确的图标格式和加载方式

## 实现步骤
1. 修改 `package.json` 添加 `build/**/*` 到 files 配置
2. 修改 `src/main/main.ts` 中的图标路径逻辑
3. 重新打包测试 Windows 应用
