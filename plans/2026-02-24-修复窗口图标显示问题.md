# 修复窗口图标显示问题

## 问题描述
在 Windows 打包后的应用中，窗口标题栏和任务栏显示的是通用文档图标，而不是自定义的应用图标。

## 问题分析
1. 当前代码在 `main.ts` 中设置了 `icon` 参数，但路径可能不正确
2. 开发模式下图标路径与打包后的路径不同
3. **关键问题（已确认）**：图标文件被打包进 `app.asar` 内部，而 `nativeImage.createFromPath()` 需要访问 asar 外部的真实文件
4. 日志显示：
   - 应用路径：`C:\Users\...\resources\app.asar`
   - 尝试加载：`C:\Users\...\resources\build\icon.ico`
   - Icon exists: false（文件不在 asar 外部）

## 解决方案
1. 使用 `extraResources` 配置将 `build` 目录放在 asar 外部
2. 修改 `src/main/main.ts` 中的图标路径逻辑，尝试多个可能的路径
3. 添加日志输出到文件，便于调试
4. 使用 `nativeImage` API 加载图标
5. 为不同平台设置正确的图标格式和加载方式

## 实现步骤
1. 修改 `package.json`，使用 `extraResources` 替代 `files` 中的 `build/**/*`
2. 修改 `src/main/main.ts` 中的图标路径逻辑，添加多个备选路径
3. 添加日志输出功能
4. 重新打包测试 Windows 应用
