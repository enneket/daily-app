# 整理版本文档

## 问题描述

当前文档中存在版本号混乱的问题：
- 文档版本号（v1.2.x）超过了软件实际版本（v1.0.2）
- 不符合语义化版本（Semantic Versioning）规范
- 用户可能会困惑

## 语义化版本规范

版本格式：`主版本号.次版本号.修订号` (MAJOR.MINOR.PATCH)

- **主版本号（MAJOR）**：不兼容的 API 修改
- **次版本号（MINOR）**：向下兼容的功能性新增
- **修订号（PATCH）**：向下兼容的问题修正

## 版本映射关系

### 实际发布版本
- v1.0.0 - 正式版（首次发布）
- v1.0.1 - 修复版（Bug 修复和小改进）
- v1.0.2 - 修复版（图标修复 + 单实例锁定）

### 开发文档版本（v1.1.x - v1.2.x）
这些是开发过程中的增量更新记录，应该归档到对应的发布版本中。

## 整理方案

### 1. 创建变更日志（CHANGELOG.md）
将所有 v1.1.x 和 v1.2.x 的更新整合到 CHANGELOG.md 中，按发布版本组织。

### 2. 重命名开发文档
将 v1.1.x 和 v1.2.x 文档移动到 `docs/changelog/` 目录，作为详细的变更记录。

### 3. 保留主要版本文档
- v1.0.0正式版发布.md
- v1.0.1版本发布.md
- v1.0.2版本发布.md

### 4. 更新文档索引
更新 docs/README.md，清晰说明版本体系。

## 实施步骤

1. 创建 CHANGELOG.md
2. 创建 docs/changelog/ 目录
3. 移动开发文档到 changelog 目录
4. 更新 docs/README.md
5. 更新主 README.md 的版本历史
