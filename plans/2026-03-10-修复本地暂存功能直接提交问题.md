# 修复本地暂存功能直接提交问题

## 问题描述

用户反馈本地暂存功能仍有问题，提交时还是会直接提交到 GitHub，而不是按预期暂存到本地。

## 问题分析

### 可能的原因

1. **shouldAutoPush() 逻辑问题**：条件判断可能仍然有误
2. **初始化问题**：某些情况下 `lastPushTime` 或 `pendingCommits` 初始化不正确
3. **缓存逻辑问题**：从 GitHub 获取内容时可能触发了意外的推送条件
4. **定时检查干扰**：定时检查可能在不合适的时机触发推送

### 当前实现检查

让我检查当前的 `shouldAutoPush()` 方法：

```typescript
private shouldAutoPush(): boolean {
  const pendingCommits = this.store.get('pendingCommits') || 0;
  const lastPushTime = this.store.get('lastPushTime');
  const now = Date.now();
  
  // 策略 1: 累积满 10 个 commit
  if (pendingCommits >= this.BATCH_SIZE) {
    console.log(`累积 ${pendingCommits} 个提交，触发自动推送到 GitHub`);
    return true;
  }
  
  // 策略 2: 有未提交的内容且距离上次提交超过 4 小时
  // 注意：如果从未推送过（lastPushTime 为 undefined），则不触发定时推送
  if (pendingCommits > 0 && lastPushTime && (now - lastPushTime) >= this.AUTO_PUSH_INTERVAL) {
    console.log(`距离上次推送已超过 4 小时，触发自动推送到 GitHub`);
    return true;
  }
  
  return false;
}
```

### 潜在问题点

1. **BATCH_SIZE 值**：如果 `BATCH_SIZE` 不是 10，可能导致意外触发
2. **pendingCommits 计算**：计数可能在某些情况下不准确
3. **lastPushTime 存在性**：可能在某些情况下意外设置了值
4. **AUTO_PUSH_INTERVAL 值**：时间间隔可能设置错误

### 2. 修复条件判断

如果发现逻辑问题，可能需要调整：

```typescript
private shouldAutoPush(): boolean {
  const pendingCommits = this.store.get('pendingCommits') || 0;
  const lastPushTime = this.store.get('lastPushTime');
  const now = Date.now();
  
  // 添加更严格的检查
  console.log('shouldAutoPush 检查:', {
    pendingCommits,
    lastPushTime,
    batchSize: this.BATCH_SIZE,
    interval: this.AUTO_PUSH_INTERVAL
  });
  
  // 策略 1: 累积满 10 个 commit
  if (pendingCommits >= this.BATCH_SIZE) {
    console.log(`触发批量提交: ${pendingCommits} >= ${this.BATCH_SIZE}`);
    return true;
  }
  
  // 策略 2: 有未提交的内容且距离上次提交超过 4 小时
  if (pendingCommits > 0 && lastPushTime) {
    const timeDiff = now - lastPushTime;
    if (timeDiff >= this.AUTO_PUSH_INTERVAL) {
      console.log(`触发定时提交: ${timeDiff}ms >= ${this.AUTO_PUSH_INTERVAL}ms`);
      return true;
    }
  }
  
  console.log('保持本地暂存');
  return false;
}
```
