# 优化页面切换时的数据加载策略

## 问题描述

从设置页面切换回主页面时，今日内容会重新加载，这是不必要的，因为数据没有变化。

## 问题分析

### 当前实现问题

1. **组件重新挂载**：每次页面切换时 `MainEditor` 组件都会重新挂载
2. **useEffect 重复执行**：组件挂载时 `useEffect` 会重新执行
3. **不必要的数据加载**：即使数据没有变化也会重新加载

### 根本原因

状态管理在子组件中，页面切换时组件销毁重建，导致状态丢失。

## 解决方案

### 方案 1：状态提升（推荐）

将今日内容和提交状态提升到 `App.tsx` 中管理：

```typescript
// App.tsx
function App() {
  const [todayReport, setTodayReport] = useState('');
  const [commitStatus, setCommitStatus] = useState<CommitStatus>({...});
  const [dataLoaded, setDataLoaded] = useState(false);
  
  // 只在应用启动时加载一次数据
  useEffect(() => {
    if (!dataLoaded) {
      loadInitialData();
    }
  }, [dataLoaded]);
  
  return (
    <div>
      {showSettings ? (
        <Settings ... />
      ) : (
        <MainEditor 
          todayReport={todayReport}
          commitStatus={commitStatus}
          onDataUpdate={handleDataUpdate}
          ...
        />
      )}
    </div>
  );
}
```

### 方案 2：缓存优化

在 `MainEditor` 中添加缓存检查，避免重复加载相同数据。

### 方案 3：使用 React Context

创建全局状态管理，避免 prop drilling。

## 实施计划

采用方案 1（状态提升），因为：
1. 简单直接，易于理解
2. 避免了不必要的网络请求
3. 提升用户体验
4. 代码结构更清晰

## 验收标准

- ✅ 从设置切换回主页面时，今日内容不重新加载
- ✅ 用户提交日报后，内容正确更新
- ✅ 跨日检查正常工作
- ✅ 页面切换速度更快
