# 优化今日已提交内容显示逻辑

## 问题描述

当前的"今日已提交内容"显示逻辑存在问题：
- 在第二天没有新增内容的情况下，仍然显示昨天的内容
- 用户期望今天没有内容时显示为空
- 需要在程序启动时或每天 0 点之后刷新显示

## 当前实现分析

### 现有逻辑
```typescript
// MainEditor.tsx
const loadTodayReport = async () => {
  try {
    const result = await window.electronAPI.getTodayReport();
    if (result.success && result.content) {
      setTodayReport(result.content);
    }
  } catch (error) {
    console.error('Failed to load today report:', error);
  }
};
```

### 问题分析
1. **缓存问题**：本地缓存可能包含昨天的内容
2. **日期判断缺失**：没有检查内容是否属于今天
3. **刷新机制缺失**：没有定时刷新或跨日刷新机制

## 解决方案

### 1. 后端优化：严格按日期过滤

修改 `getTodayReport()` 方法，确保只返回今天的内容：

```typescript
// github-service.ts
async getTodayReport(): Promise<string> {
  const today = new Date();
  const todayDateString = today.toLocaleDateString('zh-CN');
  const filePath = this.generateFilePath(today);
  
  // 1. 先检查本地缓存
  const cachedReport = this.store.get('cachedReport');
  if (cachedReport && cachedReport.filePath === filePath) {
    return this.filterTodayContent(cachedReport.content, todayDateString);
  }
  
  // 2. 从 GitHub 获取
  try {
    const { data } = await this.octokit.repos.getContent({
      owner: this.config.repoOwner,
      repo: this.config.repoName,
      path: filePath,
      ref: this.config.branch,
    });
    
    if ('content' in data) {
      const content = Buffer.from(data.content, 'base64').toString();
      return this.filterTodayContent(content, todayDateString);
    }
    return '';
  } catch (error) {
    return '';
  }
}

// 新增方法：过滤今天的内容
private filterTodayContent(content: string, todayDateString: string): string {
  const lines = content.split('\n');
  const todayLines: string[] = [];
  let isInTodaySection = false;
  
  for (const line of lines) {
    // 检查是否是今天的日期标题
    if (line.includes(todayDateString)) {
      isInTodaySection = true;
      todayLines.push(line);
      continue;
    }
    
    // 检查是否遇到其他日期（结束今天的内容）
    if (line.match(/^\# \d{4}\/\d{1,2}\/\d{1,2}/) && !line.includes(todayDateString)) {
      isInTodaySection = false;
      continue;
    }
    
    // 如果在今天的内容区域，添加到结果
    if (isInTodaySection) {
      todayLines.push(line);
    }
  }
  
  return todayLines.join('\n').trim();
}
```

### 2. 前端优化：添加定时刷新

```typescript
// MainEditor.tsx
useEffect(() => {
  loadTodayReport();
  loadCommitStatus();
  
  // 每分钟更新一次状态
  const statusInterval = setInterval(loadCommitStatus, 60000);
  
  // 每小时检查一次是否需要刷新今日内容（跨日检测）
  const refreshInterval = setInterval(() => {
    const now = new Date();
    const lastRefresh = localStorage.getItem('lastTodayReportRefresh');
    const lastRefreshDate = lastRefresh ? new Date(lastRefresh) : null;
    
    // 如果是新的一天，刷新今日内容
    if (!lastRefreshDate || 
        now.toDateString() !== lastRefreshDate.toDateString()) {
      loadTodayReport();
      localStorage.setItem('lastTodayReportRefresh', now.toISOString());
    }
  }, 60 * 60 * 1000); // 每小时检查一次
  
  return () => {
    clearInterval(statusInterval);
    clearInterval(refreshInterval);
  };
}, []);
```

### 3. 启动时刷新

```typescript
// MainEditor.tsx
useEffect(() => {
  // 启动时立即检查并刷新
  const now = new Date();
  const lastRefresh = localStorage.getItem('lastTodayReportRefresh');
  const lastRefreshDate = lastRefresh ? new Date(lastRefresh) : null;
  
  if (!lastRefreshDate || 
      now.toDateString() !== lastRefreshDate.toDateString()) {
    // 新的一天，清空显示并重新加载
    setTodayReport('');
    loadTodayReport();
    localStorage.setItem('lastTodayReportRefresh', now.toISOString());
  } else {
    // 同一天，正常加载
    loadTodayReport();
  }
  
  loadCommitStatus();
  
  // ... 其他定时器逻辑
}, []);
```

```typescript
const isSameDay = (date1: Date, date2: Date): boolean => {
  return date1.toDateString() === date2.toDateString();

// 检查内容是否属于今天
const isContentFromToday = (content: string): boolean => {
  const today = new Date().toLocaleDateString('zh-CN');
  return content.includes(today);
};
```

### 内容过滤算法
```typescript
// 按日期标题分割内容
const sections = content.split(/(?=^# \d{4}\/\d{1,2}\/\d{1,2})/m);

// 找到今天的部分
const todaySection = sections.find(section => 
  section.includes(todayDateString)
);

return todaySection || '';
```

## 风险评估

### 技术风险
- **时区问题**：需要确保日期比较使用本地时区
- **格式兼容**：日期格式需要与现有内容兼容

### 用户体验风险
- **内容丢失感知**：用户可能误以为内容丢失
- **刷新频率**：过于频繁的检查可能影响性能

## 缓解措施

### 1. 时区处理
```typescript
// 使用本地时区

const todayString = today.toLocaleDateString('zh-CN');
```

- 使用 localStorage 避免重复检查
- 合理的检查间隔（1小时）
- 只在必要时刷新
