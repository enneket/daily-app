# 修复设置页面切换空白问题

## 问题描述

用户反馈从设置页面切换回主页面时会有短暂的空白，影响用户体验。

## 问题分析

### 可能的原因

1. **异步数据加载**：切换回主页面时需要重新加载今日内容和提交状态
2. **状态重置**：页面切换时状态被清空，需要时间重新获取
3. **渲染延迟**：组件重新渲染时存在延迟
4. **网络请求阻塞**：API 调用阻塞了 UI 渲染

### 当前实现分析

让我检查主应用的页面切换逻辑和数据加载时机。

## 解决方案

### 1. 添加加载状态
在数据加载期间显示加载指示器，而不是空白页面。

### 2. 预加载数据
在设置页面时预先加载主页面需要的数据。

### 3. 缓存优化
利用已有的缓存数据，避免不必要的重新加载。

### 4. 渐进式加载
分步骤加载内容，先显示基本界面，再加载数据。

## 实施步骤

1. 检查当前的页面切换和数据加载逻辑
2. 添加加载状态管理
3. 优化数据加载时机
4. 测试用户体验改进

## 验收标准

- ✅ 从设置切换回主页面无明显空白
- ✅ 加载过程有适当的视觉反馈
- ✅ 数据加载不阻塞 UI 渲染
- ✅ 整体切换体验流畅
